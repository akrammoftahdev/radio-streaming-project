# Project Structure Addendum
**Date:** May 25, 2026

This document outlines the structural changes to the repository required to support the transition to a true Native Mobile App (React Native), Internationalization (i18n), and Audio DSP filters.

## 1. Monorepo Expansion (Native App Integration)
The current structure has two primary services (`frontend` and `backend-audio`). We will introduce a third primary package for the native mobile application.

```text
radio-streaming-project/
├── frontend/             # Existing Next.js Web App
├── backend-audio/        # Existing Node.js WebSocket & FFmpeg server
└── mobile-app/           # [NEW] React Native (Expo) Native Mobile App
    ├── app/              # Expo Router (Screens: Login, PreFlight, Studio)
    ├── components/       # Native UI Components (react-native-reanimated, etc.)
    ├── core/             # API clients, auth state, WebSockets
    ├── audio/            # Native audio recording logic & DSP filters
    └── assets/           # App icons, splash screens, fonts
```

*Note: The old `/root/android-app/` generated by Bubblewrap on the VPS will be deprecated and replaced by the actual compiled outputs from the `mobile-app` React Native codebase.*

## 2. Internationalization (i18n) Structure
To support Arabic, English, French, Spanish, German, and Portuguese, we will implement localization dictionaries in the `frontend` (and subsequently `mobile-app`).

```text
frontend/
├── messages/             # [NEW] Translation dictionaries
│   ├── en.json           # English (Default)
│   ├── ar.json           # Arabic
│   ├── fr.json           # French
│   ├── es.json           # Spanish
│   ├── de.json           # German
│   └── pt.json           # Portuguese
├── src/
│   ├── i18n.ts           # [NEW] next-intl configuration
│   ├── middleware.ts     # [MODIFIED] Update Next.js middleware to handle locale routing (e.g., /en/, /ar/)
│   └── app/
│       └── [locale]/     # [NEW] Wrap existing app routes in a locale dynamic segment
```

## 3. Audio Filter (DSP) Integration
The microphone filters will be implemented on the client side before the audio is sent to the WebSocket.

### Web Implementation
```text
frontend/src/app/studio/
├── core/
│   └── AudioProcessor.ts # [NEW] Handles AudioContext, GainNode, DynamicsCompressorNode, BiquadFilterNode (EQ), and custom ScriptProcessor/AudioWorklet for Noise Gating.
```

### Native Mobile Implementation
```text
mobile-app/audio/
├── NativeAudioProcessor.ts # [NEW] Utilizes native mobile audio bridging libraries (e.g., react-native-audio-record or Expo AV) with OS-level audio enhancements (Acoustic Echo Canceler, Noise Suppressor).
```
