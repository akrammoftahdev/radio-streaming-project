## 4. Monorepo vs VPS Structure

### Local (Monorepo)
```
radio_streaming_project/
├── frontend/          # Next.js 16, React 19
├── backend-audio/     # Liquidsoap + Express bridge
├── mobile-app/        # React Native / Expo
├── infra/             # Infrastructure scripts
├── deploy/            # Deployment scripts
└── project-knowledge-base/
```

### VPS (/var/www/egonair/frontend/)
```
/var/www/egonair/frontend/    ← PM2 cwd, `npm start` runs here
├── src/                      # Source files (synced via rsync)
├── public/                   # Static assets
├── prisma/                   # Schema + seed scripts
├── package.json              # Next.js frontend only
├── tsconfig.json             # Frontend-only config
├── next.config.ts            # Next.js config
├── .next/                    # Build output (generated by `npm run build`)
└── node_modules/             # Dependencies
```

> **CRITICAL:** The VPS does NOT have the full monorepo. It only has the `frontend/` directory content flattened to the root. Never `git reset --hard origin/main` — it will replace the flat structure with the full monorepo and break everything.
