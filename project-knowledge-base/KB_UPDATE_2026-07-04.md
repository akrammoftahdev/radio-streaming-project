# Knowledge Base Update: July 4, 2026

## CRITICAL: VPS Production Architecture (No Google Cloud)

A major error occurred previously because it was assumed that the project was hosted on Google Cloud (using GCS buckets and Cloud Run) due to leftover Google Cloud templates in the deployment scripts.

**This is strictly a VPS-hosted architecture.**
There are no external cloud services (AWS, GCP, etc.) for data storage or compute.

### Current Production Environment:
*   **Host**: Dedicated VPS (IP: `195.35.48.184`)
*   **Domain**: `studio.egonair.com`
*   **Services**:
    *   `egonair-frontend` (Next.js) running on port `3000`.
    *   `egonair-audio` (Node.js/WebSocket backend) running on port `4001`.
*   **Process Management**: Both services are managed by PM2 running as `root`.
*   **Filesystem**: Both services share the local filesystem for recording storage.

### Recordings Configuration
Recordings must be saved to a persistent, shared directory on the VPS so that the backend can write them and the frontend can serve them.

*   **Correct Path**: `/mnt/recordings`
*   **Frontend `.env`**: `RECORDINGS_BASE_DIR="/mnt/recordings"`
*   **Backend `.env`**: `RECORDINGS_BASE_DIR=/mnt/recordings`

> **Note on past issue:** A deployment overwrote the backend's `.env` with a template that pointed to `/tmp/recordings`. This caused the backend to save new `.mp3` files in `/tmp/recordings` while the frontend looked for them in `/mnt/recordings`, resulting in a "File not found" error in the dashboard. This has been resolved.

### Shoutcast Integration
The backend handles real-time audio broadcasting using FFmpeg to send streams to SHOUTcast.
*   **FFmpeg Path on VPS**: `/usr/bin/ffmpeg`
*   **Live Broadcast Toggle**: `ENABLE_SHOUTCAST_LIVE=true` must remain enabled in the backend `.env`.

### WebSocket and API URLs
*   **Frontend WebSocket URL**: `NEXT_PUBLIC_WS_URL="wss://studio.egonair.com/audio"`
*   **Backend Token Validation**: The backend validates tokens against the frontend via `NEXT_VALIDATE=http://127.0.0.1:3000/api/internal/audio-token/validate`.

**Future Agents:** NEVER modify the `.env` files to point to Google Cloud storage buckets or Cloud Run instances. Always verify the live configuration on the VPS before replacing environment files.
