# KB Update - 2026-05-21

## 1. Recordings 404 Issue & GCS FUSE Resolution
**Symptom:** Presenters attempting to download completed recordings received a 404 Not Found from the `frontend` API.
**Root Cause:** The `backend-audio` Cloud Run service was saving the recordings to its own ephemeral local disk. The `frontend` Cloud Run service was searching its own separate ephemeral local disk.
**Solution:** 
- Created a Google Cloud Storage bucket: `egonair-recordings-shared`
- Granted `roles/storage.objectAdmin` to `egonair-frontend@egonair-stream-prod.iam.gserviceaccount.com`
- Mounted the bucket to BOTH `backend-audio` and `frontend` using Cloud Storage FUSE at `/mnt/recordings`.
- Updated both environments with `RECORDINGS_BASE_DIR=/mnt/recordings`.

## 2. iOS Background Audio Disconnects
**Symptom:** When broadcasting from an iPhone, locking the screen or backgrounding the browser instantly killed the broadcast.
**Root Cause:** iOS explicitly terminates `getUserMedia` microphone tracks the moment Safari or Chrome loses active foreground focus to preserve battery and privacy. Apple strictly prohibits websites from running background audio capture.
**Workaround (Short Term):** Implemented `nosleep.js` in `studio-ui.tsx`. When the presenter clicks "Connect", a silent, invisible HTML5 `<video>` element loops continuously. iOS detects the video playback and prevents the screen from automatically locking.
**Solution (Long Term):** See `NATIVE_APP_ROADMAP_2026-05-21.md`.
