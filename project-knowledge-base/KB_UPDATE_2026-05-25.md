# Knowledge Base Update: May 25, 2026

## Overview of Strategic Shift
After extensive testing of the Bubblewrap/TWA (Trusted Web Activity) Android wrapper, we identified critical limitations regarding background audio processing. The Android OS suspended the WebView's `MediaRecorder` when the screen locked or app lost focus, resulting in 0-byte streams being sent to SHOUTcast.

As a result, we have decided to abandon the WebView wrapper approach for mobile and pivot to a **100% clean Native Mobile App** architecture using React Native (Expo). This ensures reliable background microphone access via foreground services and allows for proper Apple App Store deployment.

## New Planned Features
1. **True Native Mobile Apps (React Native / Expo):**
   - Direct hardware integration for the microphone.
   - Background audio execution (Foreground Services on Android, Audio Session on iOS).
   - Will integrate directly with our existing Node.js WebSocket backend and PostgreSQL database.
   - Will natively enforce our existing multi-role timer logic (Direct DJ vs Multi-Station vs Single-Station).

2. **Internationalization (i18n) & Multilingual Support:**
   - The web platform (Next.js) will be translated into 6 languages: Arabic, English, French, Spanish, German (Deutsch), and Portuguese.
   - Infrastructure: Will use `next-intl` (or similar) for translation dictionaries.
   - The UI will dynamically switch layouts to RTL (Right-To-Left) when Arabic is selected, preserving the glassmorphism aesthetics.

3. **Microphone Audio Filters:**
   - As discussed in prior sessions, we will implement `AudioContext` DSP (Digital Signal Processing) filters before the audio hits the `MediaRecorder`.
   - Planned filters: Gain control (Volume), Equalizer (EQ), Compressor (to normalize volume spikes), and Noise Gate (to eliminate background hum when not speaking).
